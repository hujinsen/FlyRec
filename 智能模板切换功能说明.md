# FlyRec 智能模板切换功能

## 功能概述

FlyRec 现在支持根据当前鼠标焦点所在的应用程序自动切换提示词模板，让语音识别结果更符合使用场景。

## 支持的应用类型与“场景”

### 🗣️ 聊天场景
**适用应用：**
- 微信 (wechat.exe)
- QQ (qq.exe, qqscm.exe)
- 钉钉 (dingtalk.exe)
- Microsoft Teams (teams.exe)
- Slack (slack.exe)
- Telegram (telegram.exe)
- Discord (discord.exe)

**场景定位：** 保留用户原始表达意图，对明显口语赘词做轻量清理与标点补全；不会**生成**新的聊天回复，不替你“代聊”，仅规范你说的话。

### 📧 邮件场景
**适用应用：**
- Outlook (outlook.exe)
- Thunderbird (thunderbird.exe)
- Foxmail (foxmail.exe)
- 网易邮箱大师 (mailmaster.exe)

**场景定位：** 在尽量不改变语义的前提下，进行结构化与礼貌性润色（称呼/结尾可补齐）。不会虚构内容。

### 💻 代码场景
**适用应用：**
- Visual Studio Code (code.exe)
- Visual Studio (devenv.exe)
- PyCharm (pycharm64.exe)
- IntelliJ IDEA (idea64.exe)
- Notepad++ (notepad++.exe)
- Sublime Text (sublime_text.exe)
- Atom (atom.exe)
- WebStorm (webstorm64.exe)
- PHPStorm (phpstorm64.exe)
- CodeBuddy (codebuddy.exe)

**场景定位：** 将你的口述需求转写为更清晰的“需求描述”或伪代码样式（当前版本不自动补全真正的长代码实现，只规范表达）。

### 📝 默认场景
**适用应用：**
- 记事本 (notepad.exe)
- 写字板 (wordpad.exe)
- Microsoft Word (winword.exe)
- Excel (excel.exe)
- PowerPoint (powerpnt.exe)

**场景定位：** 去除常见语气词/口头停顿，补全基本标点，保持原意输出。

## 智能检测机制

### 1. 进程名检测
系统会检测当前活动窗口的进程名，如果匹配预设的应用列表，直接切换对应模板。

### 2. 窗口标题关键词检测
如果进程名未匹配，系统会分析窗口标题中的关键词：
- 聊天关键词：微信、wechat、qq、钉钉
- 邮件关键词：邮件、mail、outlook、邮箱
- 代码关键词：code、visual studio、pycharm、idea

### 3. 实时切换
每次开始语音识别时，系统会实时检测当前活动窗口并选择最合适的模板。

## 使用方法

### 启用智能模板切换
1. 打开 FlyRec 应用
2. 进入"设置"页面
3. 勾选"智能模板切换（根据当前应用自动选择模板）"选项

### 手动场景选择
如果需要手动指定模板，可以：
1. 取消勾选智能模板切换选项
2. 在"文本处理模板"中选择固定模板

## 测试方法

使用提供的测试脚本验证功能：
```bash
python test_smart_template.py
```

测试脚本会显示：
- 当前活动窗口的进程名
- 窗口标题
- 推荐的模板类型

## 语言模式与提示词优先级

语言模式（中文 / 英语）只影响输出语言，不改变“是否生成新内容”的原则。提示词采用如下回退顺序：
1. 当前场景自定义提示词（若存在并非空）
2. 全局默认提示词（中文模式）
3. 内置内核默认（确保永不为空）
4. 英语模式：在上述提示词后附加英文输出指令

## 技术依赖

- `psutil`: 用于获取进程信息
- `pywin32`: 用于Windows窗口API调用

## 注意事项

1. 智能模板切换功能仅在Windows系统上可用
2. 需要安装psutil和pywin32依赖包
3. 如果依赖包未安装，系统会自动回退到手动模板选择模式
4. 可以随时在设置中开启或关闭智能模板切换功能

## 扩展性

可以通过修改`app_template_mapping`字典来添加更多应用程序的支持：

```python
self.app_template_mapping = {
    'your_app.exe': '模板类型',
    # 添加更多应用...
}
```

## 故障排查

| 问题 | 可能原因 | 建议处理 |
|------|----------|----------|
| 场景没有自动切换 | 当前窗口进程名不在映射，或缺少权限 | 确认是否安装 `pywin32`，前台窗口是否真实聚焦；可手动勾选/取消智能切换验证 | 
| 输出仍是原始口头语 | 模板是“轻量润色”策略 | 这是正常设计；若需更强改写，可自定义提示词 | 
| 邮件输出没有称呼/结尾 | 口述内容过短 | 添加“邮件”关键短语或在提示词中强制结构 | 
| 代码场景没生成代码 | 当前版本不负责自动代码生成 | 后续版本可引入可选“生成”开关 |

## 结语

智能模板切换让 FlyRec 在“准确转写 + 适度润色”之间取得平衡：**不替你编造，不代你回复**，而是把你说的内容整理得更易直接使用。欢迎根据实际工作流调整提示词与场景映射，让它更贴合你的个人习惯。
